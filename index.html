Kiến thức về Git cơ bản

Git là gì?

Git là một hệ thống quản lý phiên bản phân tán (Distributed Version Control System - DVCS) mã nguồn mở, được sử dụng rộng rãi để theo dõi thay đổi trong mã nguồn phần mềm. Nó giúp lập trình viên lưu lịch sử thay đổi, quay lại phiên bản cũ, làm việc nhóm hiệu quả mà không ảnh hưởng đến mã chính.

Cài đặt Git

  • Trên Windows: Tải từ trang chính thức https://git-scm.com/download/win. Chạy file cài đặt, giữ các tùy chọn mặc định và hoàn tất. Sau cài đặt, bạn có Git Bash để sử dụng lệnh.
  • Trên macOS: Sử dụng Homebrew (brew install git) hoặc tải từ trang chính thức.
  • Trên Linux (Ubuntu/Debian): Chạy lệnh sudo apt update && sudo apt install git.
  • Kiểm tra cài đặt: Mở terminal/cmd và gõ git --version.

Cấu hình cơ bản

Sau khi cài đặt, thiết lập thông tin cá nhân (chỉ cần làm một lần):

git config --global user.name "Tên của bạn"
git config --global user.email "email@domain.com"

Các lệnh Git cơ bản

Dưới đây là các lệnh phổ biến nhất cho người mới bắt đầu. Workflow cơ bản: Thay đổi file → add → commit → push (nếu có remote).

Lệnh Mô tả Ví dụ
git init Khởi tạo repository mới trong thư mục hiện tại. git init
git clone <url> Sao chép repository từ remote (ví dụ GitHub). git clone https://github.com/user/repo.git
git status Kiểm tra trạng thái thay đổi (file nào modified, staged...). git status
git add <file> Thêm file thay đổi vào staging area (chuẩn bị commit). git add index.html
Hoặc git add . (thêm tất cả)
git commit -m "message" Lưu thay đổi vào lịch sử (commit) với mô tả. git commit -m "Sửa lỗi login"
git log Xem lịch sử commit. git log --oneline (ngắn gọn)
git branch Liệt kê các branch. Tạo branch mới: git branch <tên>. git branch feature/new
git checkout <branch> Chuyển sang branch khác.
Tạo và chuyển: git checkout -b <tên>.		 git checkout main
git checkout -b develop
git merge <branch> Hợp nhất branch vào branch hiện tại. git merge feature/new
git pull Lấy thay đổi mới từ remote và merge vào local. git pull origin main
git push Đẩy commit lên remote. git push origin main
git remote add origin <url> Kết nối repository local với remote (GitHub/GitLab...). git remote add origin https://github.com/user/repo.git
git diff Xem sự khác biệt thay đổi chưa staged. git diff

Workflow cơ bản cho dự án mới

  1. Tạo repository trên GitHub/GitLab.
  2. Clone về local: git clone <url>.
  3. Tạo branch mới nếu cần: git checkout -b ten-branch.
  4. Thay đổi file → git add . → git commit -m "Mô tả".
  5. Đẩy lên: git push origin ten-branch.
  6. Tạo Pull Request trên web để merge.

Hình minh họa

Lưu ý thường gặp

  • Tránh commit file nhạy cảm (password, key): Tạo file .gitignore để bỏ qua (ví dụ: .env, node_modules/).
  • Xung đột (conflict): Khi merge/pull, Git sẽ báo nếu có xung đột → Chỉnh sửa file thủ công rồi commit lại.
  • Undo thay đổi: git restore <file> (bỏ thay đổi chưa add), hoặc git reset --hard (cẩn thận, xóa hết thay đổi).

Nếu bạn mới bắt đầu, hãy thực hành trên một thư mục test. Khuyến nghị đọc thêm hướng dẫn đơn giản tại: https://rogerdudler.github.io/git-guide/index.vi.html (bản tiếng Việt).

Trang này lưu toàn bộ nội dung hướng dẫn Git cơ bản. Bạn có thể lưu file này về máy và mở bằng trình duyệt để xem offline.
